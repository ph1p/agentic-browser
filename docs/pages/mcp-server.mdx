# MCP Server

The MCP (Model Context Protocol) server exposes agentic-browser as native tools in AI coding assistants. Instead of shelling out to CLI commands, the assistant calls structured tools directly — no JSON parsing, no shell escaping, no subprocess overhead.

## Setup

### Automatic

```bash
npx agentic-browser setup
```

Detects Claude Code and Cursor, then writes the config for you.

### Manual: Claude Code

Add to `.mcp.json` in your project root:

```json
{
  "mcpServers": {
    "agentic-browser": {
      "command": "npx",
      "args": ["agentic-browser", "mcp"]
    }
  }
}
```

### Manual: Cursor

Add to `.cursor/mcp.json` in your project root:

```json
{
  "mcpServers": {
    "agentic-browser": {
      "command": "npx",
      "args": ["agentic-browser", "mcp"]
    }
  }
}
```

### Use your existing Chrome

To control a Chrome you already have open (with all your sessions and cookies), launch Chrome with `--remote-debugging-port=9222` and add `AGENTIC_BROWSER_CDP_URL` to your MCP config:

```json
{
  "mcpServers": {
    "agentic-browser": {
      "command": "npx",
      "args": ["agentic-browser", "mcp"],
      "env": {
        "AGENTIC_BROWSER_CDP_URL": "http://127.0.0.1:9222"
      }
    }
  }
}
```

Alternatively, launch a fresh Chrome with your default profile (Chrome must be closed first):

```json
{
  "mcpServers": {
    "agentic-browser": {
      "command": "npx",
      "args": ["agentic-browser", "mcp"],
      "env": {
        "AGENTIC_BROWSER_USER_PROFILE": "default"
      }
    }
  }
}
```

Run headless with a custom user-agent:

```json
{
  "mcpServers": {
    "agentic-browser": {
      "command": "npx",
      "args": ["agentic-browser", "mcp"],
      "env": {
        "AGENTIC_BROWSER_HEADLESS": "true",
        "AGENTIC_BROWSER_USER_AGENT": "MyBot/1.0"
      }
    }
  }
}
```

### Other MCP Clients

Any MCP-compatible client can connect via stdio transport:

```bash
npx agentic-browser mcp
```

The server communicates over stdin/stdout using the MCP JSON-RPC protocol.

## Tools

### `browser_start_session`

Start a Chrome browser session. Most workflows do not need to call this explicitly because browser tools auto-start a session when needed.

**Parameters:** None

**Returns:** Session object with `sessionId`, `status`, `startedAt`. (`authTokenRef` is omitted from MCP responses.)

---

### `browser_navigate`

Navigate the browser to a URL. After successful navigation, the server also attempts a best-effort cookie-banner dismissal and includes the result when something was dismissed.

| Parameter   | Type   | Required | Description                    |
| ----------- | ------ | -------- | ------------------------------ |
| `url`       | string | Yes      | The URL to navigate to         |
| `sessionId` | string | No       | Uses active session if omitted |

**Returns:** `resultStatus`, `resultMessage`, `currentUrl`, and optional `cookieBanner`. On failure, also includes `errorCode`.

---

### `browser_interact`

Interact with a page element or perform browser-level actions.

**Element actions:** `click`, `type`, `press`, `waitFor`, `scroll`, `hover`, `select`, `toggle`

**Navigation actions:** `goBack` (browser back), `goForward` (browser forward), `refresh` (reload page)

**Dialog action:** `dialog` (handle JS alert/confirm/prompt)

| Parameter           | Type     | Required | Description                                                                            |
| ------------------- | -------- | -------- | -------------------------------------------------------------------------------------- |
| `action`            | string   | Yes      | One of the actions listed above                                                        |
| `selector`          | string   | No       | CSS selector for the target element                                                    |
| `fallbackSelectors` | string[] | No       | Backup CSS selectors tried if the primary selector fails (from `browser_get_elements`) |
| `text`              | string   | No       | Text to type (for `type`), or `"dismiss"` to dismiss a dialog (for `dialog`)           |
| `key`               | string   | No       | Key to press (for `press` action, e.g. `"Enter"`)                                      |
| `value`             | string   | No       | Option value (for `select`), or prompt text (for `dialog`)                             |
| `scrollX`           | number   | No       | Horizontal scroll delta in pixels (for `scroll`)                                       |
| `scrollY`           | number   | No       | Vertical scroll delta in pixels (for `scroll`, positive = down)                        |
| `timeoutMs`         | number   | No       | Timeout in ms (for `waitFor` action)                                                   |
| `sessionId`         | string   | No       | Uses active session if omitted                                                         |

Navigation actions (`goBack`, `goForward`, `refresh`) and `dialog` don't require a selector.

**Returns:** `resultStatus`, `resultMessage`, and `currentUrl`. On failure, also includes `errorCode`.

**Error codes:** When `resultStatus` is `"failed"`, the `errorCode` field contains one of: `SELECTOR_NOT_FOUND`, `ELEMENT_COVERED`, `ELEMENT_NOT_EDITABLE`, `NAVIGATION_TIMEOUT`, `NAVIGATION_FAILED`, `CDP_DISCONNECTED`, `SESSION_DEAD`, `DIALOG_BLOCKING`, `TIMEOUT`, `UNKNOWN`.

---

### `browser_get_content`

Get the current page content. MCP responses are compacted for LLMs so the model can understand page structure without parsing a large raw blob first.

| Parameter   | Type                                                         | Required | Default     | Description                                                                                                 |
| ----------- | ------------------------------------------------------------ | -------- | ----------- | ----------------------------------------------------------------------------------------------------------- |
| `mode`      | `"title"` \| `"text"` \| `"html"` \| `"a11y"` \| `"summary"` | No       | `"summary"` | Extraction mode                                                                                             |
| `selector`  | string                                                       | No       |             | CSS selector to scope content                                                                               |
| `maxChars`  | number                                                       | No       |             | Max characters to return (default: 8000 for text, 10000 for a11y, 4000 for html, ignored for summary/title) |
| `sessionId` | string                                                       | No       |             | Uses active session if omitted                                                                              |

`summary` is the recommended first read because it returns a low-token overview with `title`, `headings`, `landmarks`, `primaryActions`, `inputs`, `alerts`, iframe metadata, `contentExcerpt` (up to 1500 chars of visible text), and `forms` (grouped form fields with labels, types, required status, and current values), including whether frames are same-origin or cross-origin.

Text, HTML, and accessibility content are automatically truncated at the default `maxChars` limit. When truncated, the response includes a `truncated: true` flag and a notice. Use a CSS `selector` to scope content instead of raising the limit.

**Response shape by mode:**

- `summary`: `{ mode, url, title, headings, landmarks, primaryActions, inputs, alerts, frames, totalInteractiveElements, truncatedInteractiveElements, contentExcerpt, forms }`
- `title`: `{ mode, content, truncated }`
- `text`: `{ mode, content, truncated, summaryLines, previewLines }`
- `a11y`: `{ mode, content, truncated, headings, landmarks, actions, inputs, previewLines }`
- `html`: `{ mode, content, truncated }`

---

### `browser_get_elements`

Discover interactive elements on the page. Returns selectors and fallback selectors usable with `browser_interact`. Pass `fallbackSelectors` to `browser_interact` for automatic retry when the primary selector breaks. When an element is inside an open shadow root or a same-origin iframe, the selector uses `>>>` to cross that boundary.

| Parameter     | Type     | Required | Default | Description                                                                                                                 |
| ------------- | -------- | -------- | ------- | --------------------------------------------------------------------------------------------------------------------------- |
| `roles`       | string[] | No       | all     | Filter: `"link"`, `"button"`, `"input"`, `"select"`, `"textarea"`, `"checkbox"`, `"radio"`, `"contenteditable"`, `"custom"` |
| `visibleOnly` | boolean  | No       | `true`  | Only visible elements                                                                                                       |
| `limit`       | number   | No       | `50`    | Max elements to return                                                                                                      |
| `selector`    | string   | No       |         | Scope to a subtree                                                                                                          |
| `sessionId`   | string   | No       |         | Uses active session if omitted                                                                                              |

**Returns:**

```json
{
  "elements": [
    {
      "selector": "#login-btn",
      "fallbackSelectors": ["button[aria-label=\"Login\"]", "[data-testid=\"login\"]"],
      "role": "button",
      "text": "Login"
    }
  ],
  "totalFound": 42,
  "truncated": true,
  "summary": {
    "countsByRole": {
      "button": 12,
      "link": 18
    },
    "primaryActions": [
      {
        "role": "button",
        "text": "Login",
        "selector": "#login-btn",
        "fallbackSelectors": ["button[aria-label=\"Login\"]", "[data-testid=\"login\"]"]
      }
    ]
  }
}
```

MCP responses omit `visible` (always true when `visibleOnly` is true), `actions` (derivable from `role`), `tagName` (redundant with `role`), and `enabled` when it is `true`. These fields are still available via the programmatic API.

---

### `browser_search_memory`

Search task memory for previously learned selectors, selector aliases, and interaction patterns. Results include `selectorHints` (proven selectors) and `selectorAliases` (human-readable names mapped to selectors with fallbacks).

| Parameter    | Type   | Required | Default | Description                                    |
| ------------ | ------ | -------- | ------- | ---------------------------------------------- |
| `taskIntent` | string | Yes      |         | What you want to do, e.g. `"login:github.com"` |
| `siteDomain` | string | No       |         | Domain to scope search                         |
| `limit`      | number | No       | `5`     | Max results                                    |

---

### `browser_dismiss_cookies`

Dismiss a cookie or consent banner on the current page. The tool tries fast DOM heuristics first, then falls back to the accessibility tree for banners that are harder to target.

| Parameter   | Type   | Required | Description                    |
| ----------- | ------ | -------- | ------------------------------ |
| `sessionId` | string | No       | Uses active session if omitted |

---

### `browser_screenshot`

Take a screenshot of the current page and return the image as a base64-encoded string.

| Parameter   | Type    | Required | Default  | Description                           |
| ----------- | ------- | -------- | -------- | ------------------------------------- |
| `format`    | string  | No       | `"jpeg"` | Image format: `"jpeg"` or `"png"`     |
| `quality`   | number  | No       |          | JPEG quality (0-100, ignored for PNG) |
| `fullPage`  | boolean | No       | `false`  | Capture the full scrollable page      |
| `sessionId` | string  | No       |          | Uses active session if omitted        |

**Returns:** Base64-encoded image data.

---

### `browser_stop_session`

Stop the browser session and terminate Chrome.

| Parameter   | Type   | Required | Description                    |
| ----------- | ------ | -------- | ------------------------------ |
| `sessionId` | string | No       | Uses active session if omitted |

## Session Management

The MCP server tracks a single active session. When you call `browser_start_session`, the returned `sessionId` becomes the active session. All subsequent tool calls use it automatically unless you pass a specific `sessionId`.

When the MCP client disconnects, the server automatically cleans up any active session.

## Example Workflow

A typical sequence an AI assistant would use:

1. `browser_start_session` — optional, launch Chrome explicitly
2. `browser_navigate` with `url: "https://github.com/login"` — cookie dismissal runs automatically after successful navigation
3. `browser_get_content` with `mode: "summary"` — get the low-token overview first
4. `browser_get_elements` — get selectors, fallback selectors, and summarized primary actions
5. `browser_interact` with `action: "type"`, `selector: "input[name=login]"`, `fallbackSelectors: [...]`, `text: "username"`
6. `browser_interact` with `action: "type"`, `selector: "input[name=password]"`, `text: "***"`
7. `browser_interact` with `action: "click"`, `selector: "input[type=submit]"`
8. `browser_get_content` with `mode: "text"` or `mode: "a11y"` only when the summary is not enough
9. `browser_stop_session` — done
